Key Architecture Decisions

  • Separate admin panel — Admins access /admin routes behind Clerk auth. Public site has zero admin UI.
  • Newsletter subscribers managed externally — MailerLite handles signup forms, subscriber lists, and campaigns. Admin panel does not duplicate this.
  • File viewing — Newsletter PDFs and meeting minutes open in-browser (no download required for viewing).
  • Admin login access — Small footer link on public site. Not prominent — admins know where to find it.
  • Help Center — Built into the admin panel so admins get guidance in-context where they work.
